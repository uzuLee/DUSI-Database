## 01. 목적

본 매뉴얼은 ARCHIVE-Σ 중앙 아카이브와 현장 단말기 간의 데이터 동기화 과정에서 발생하는 치명적 오류를 제어하고, **[해시 인장(Hash Seal)]** 불일치 시 관리자가 수행해야 할 수동 검증 및 강제 동기화 절차를 규정한다.

## 02. 터미널 접속 및 인가

1. (주)가온정밀시스템의 중앙 통제실 요원은 ARCHIVE-Σ-OS 접속 시 물리적 망분리 환경에서만 접근해야 한다.
2. 접속 명령어: `> central_core_connect --auth [관리자 ID] --bypass_reality_check`

## 03. 해시 검증 오류 패턴 및 대처

터미널에 아래와 같은 비정형 오류 메시지가 출력될 경우, 이는 단순한 통신 장애가 아닌 '현실의 논리적 붕괴'가 시스템에 간섭하고 있음을 의미한다.

### 오류 A: `FATAL: Reality Anchor Not Found (현실 닻 유실)`
- **원인**: 현장의 대상 구역이 이미 **[위상적 동화]**를 마치고 현실에서 떨어져 나갔음. 해당 구역의 단말기는 존재하지 않는 좌표에서 데이터를 보내고 있음.
- **조치**: 
  1. 즉시 해당 단말기의 연결을 차단 (`> disconnect_node --force`).
  2. 중앙관리국에 해당 좌표의 **[음각]** 선포 요청.

### 오류 B: `WARNING: Hash Collision with Void (공백과의 해시 충돌)`
- **원인**: 수신된 데이터가 이미 **[존재의 소급 말소]**를 당한 인원 또는 객체의 식별자와 일치함. 즉, 지워진 존재가 시스템에 다시 등록되려 함.
- **조치**:
  1. 수동으로 메모리 누수 처리 명령을 실행 (`> purge_void_memory`).
  2. 시스템에 충돌을 유발한 현장 요원의 신변을 확인 후 재사회화 회부.

### 오류 C: `ERROR: Cognitive Shield Overloaded (인지 방벽 과부하)`
- **원인**: 단말기를 조작 중인 요원의 인지가 심각하게 오염되어, 뇌파의 논리적 오류가 단말기의 전기 신호에까지 영향을 미치고 있음.
- **조치**: 
  1. 데이터 수신을 중지하고 단말기의 '논리 파쇄 장치'를 원격 폭파시킴 (`> trigger_self_destruct`).

## 04. 강제 동기화 (Force Sync)

오류에도 불구하고 공단의 행정적 진실을 유지하기 위해 데이터를 강제로 기록해야 할 경우:
`> force_sync_hash --ignore_anomaly --write_truth`
명령어를 실행한다. 

**※ 경고**: 이 명령어를 실행한 직후, 통제실 내부에서 이명, 환각, 또는 원인 모를 한기를 느낀다면 시스템의 노이즈가 터미널을 타고 역류한 것이므로, 작업자는 즉시 눈을 감고 보안팀을 호출할 것.
